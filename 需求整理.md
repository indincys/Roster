## 1. 产品定位

短视频运营工作台是一款面向短视频商业发布、带货内容运营和 AI 内容生产的跨平台桌面应用。

软件支持 macOS 和 Windows 双平台运行，且两端功能完全一致。用户可以在任一系统上完成工作空间管理、内容创作、数据库维护、封面处理、任务单生成、任务单导出、状态查看、备份和设置管理。不存在“Mac 端只能创作、Windows 端才能管理或导出”的功能限制。

软件采用本地优先的工作方式。视频、封面、图片、任务单、数据库、Skill 配置、备份和日志均围绕本地工作空间组织。软件需要兼容 OneDrive、坚果云、Dropbox 等云盘同步目录，以支持用户在 macOS 与 Windows 工作电脑之间切换。影刀 RPA 当前作为外部执行环境运行在 Windows 端，但任务单的准备、编辑、导出和状态查看可以在 macOS 或 Windows 任一端完成。

软件不是单一功能工具，而是一个可持续扩展的工作流容器。后续可以持续接入新的短视频运营工作流、AI 生成工作流、素材管理工作流和自动化任务。

## 2. 产品核心原则

1. 高频功能必须快速可达，核心入口应尽量控制在 1–2 步内完成访问。
2. 软件结构必须具备长期扩展能力，未来新增更多工作流时不能造成导航混乱。
3. 软件第一次打开时，用户应能直观找到主要功能，不依赖完整教程才能开始使用。
4. 多品牌必须彻底隔离，不同品牌的内容、风格、视频、标题、文案、Skill 和任务单互不影响。
5. 批量操作是主要交互模式，单条操作是辅助能力。
6. 多模型对比是高频能力，前期必须支持，便于在模型能力变化时快速测试和切换。
7. 本地文件、路径映射、云盘同步和 RPA 执行链路必须稳定可靠。
8. 软件 UI 以中文为主，英文主要用于变量名、日志、技术字段和导出字段可选项。

## 5. 工作空间与多品牌管理

### 5.1 工作空间定义

工作空间等同于品牌。每个品牌拥有一个独立工作空间。

不同工作空间之间的数据、内容、风格和任务单完全隔离。A 品牌的视频、标题、文案、Skill 启用配置和任务单不会被 B 品牌复用。

### 5.2 工作空间隔离范围

每个工作空间独立包含：

- 视频库
- 标签库
- 标题库
- 图片提示词库
- 图片库
- 视频文案库
- 任务单
- Skill 启用配置
- 平台账号清单
- 备份数据

### 5.3 工作空间共享范围

以下资源为全局共享：

- API key 凭证池
- Skill 池本体

Skill 池本体全局共享，但每个工作空间独立配置启用哪些 Skill。

### 5.4 工作空间内的平台账号

一个工作空间可以配置多个平台账号。同一工作空间内的多个平台账号指向同一品牌，账号之间共享该品牌的视频、标题、标签、封面、文案和图片素材。

### 5.5 协作能力

v1 不实现多人协作和共享工作空间。v1 采用独立工作实例，不同实例互不影响。

## 6. 工作空间目录结构

### 6.1 工作空间根目录

用户首次创建工作空间时选择硬盘位置。软件强烈引导用户选择 OneDrive、坚果云、Dropbox 等云盘同步目录作为工作空间根目录。

工作空间根目录采用以下结构：

```text
品牌A/
├── videos/         # 视频原文件
├── covers/         # 封面文件
├── images/         # AI 生成的图片素材
├── tasks/          # 任务单导出文件
├── skills_config/  # 当前工作空间启用哪些 Skill 的配置
└── _backup/        # 自动备份
```

Skill 本体不放在单个工作空间内。Skill 本体放在全局位置，各工作空间通过 `skills_config/` 记录启用关系。

### 6.2 视频目录

视频原文件放在工作空间的 `videos/` 目录下，并按 SKU 或款式组织，例如：

```text
videos/
├── SKU001/
│   ├── style_a.mp4
│   └── style_b.mp4
└── SKU002/
    └── style_c.mp4
```

软件不把视频文件复制进数据库或应用内部。软件只索引视频文件路径、文件名、SKU 编码、款式、品牌和辅助信息。

### 6.3 封面目录

封面文件放在独立 `covers/` 目录下，并镜像 `videos/` 目录结构，例如：

```text
covers/
├── SKU001/
│   ├── style_a__3x4.jpg
│   ├── style_a__9x16.jpg
│   └── style_b__3x4.jpg
└── SKU002/
    └── style_c__9x16.jpg
```

封面文件不直接混放在视频目录中。一个视频可以同时存在多个比例的封面。

### 6.4 任务单目录

任务单固定导出到当前工作空间的 `tasks/` 目录下，并按日期分组：

```text
品牌A/
└── tasks/
    └── 2026-05-07/
        ├── tasks.xlsx
        ├── tasks.csv
        ├── tasks.json
        ├── preflight.json
        └── status/
            ├── T20260507-001__attempt-001.json
            └── T20260507-002__attempt-001.json
```

任务单导出位置固定跟随工作空间，不提供自定义导出目录。

## 7. 跨平台路径与云盘同步

### 7.1 路径存储原则

数据库只存相对路径，不直接存 Mac 或 Windows 的绝对路径。

示例：

```text
videos/SKU001/style_a.mp4
covers/SKU001/style_a__9x16.jpg
```

### 7.2 工作空间路径映射

每个工作空间保存 Mac 根路径和 Windows 根路径两套配置。

示例：

```text
Mac 根路径：/Users/indincys/CloudSync/品牌A/
Win 根路径：D:\CloudSync\品牌A\
```

### 7.3 路径展示与导出

软件界面按当前操作系统拼接并展示绝对路径。

任务单导出时必须明确“目标执行平台”。v1 默认目标执行平台为 Windows 影刀 RPA，因此导出给 RPA 的视频路径和封面路径默认按 Windows 根路径拼接为 Windows 绝对路径。即使用户在 macOS 上生成或导出任务单，导出的 RPA 路径也应使用 Windows 根路径；如果用户在 Windows 上生成或导出任务单，软件同样按当前工作空间的 Windows 根路径生成 RPA 路径。

界面中应同时区分：

- 当前设备可访问路径：按当前系统根路径拼接，用于本机预览、校验和打开文件。
- RPA 执行路径：按目标执行平台根路径拼接，用于任务单导出。

### 7.4 路径校验

任务单导出前必须进行路径校验。校验内容包括：

- 视频文件是否存在
- 封面文件是否存在
- 路径是否可被当前系统访问
- 当前设备上的文件是否真实可读，而不是云盘占位符
- 视频文件是否能通过基础媒体校验，例如读取头部或 ffprobe 元数据
- 封面文件是否能通过基础图片校验，例如读取尺寸
- Windows 导出路径是否具备可被 RPA 识别的路径格式
- 当导出发生在 macOS 端时，软件只能完成“本机文件可读”和“Windows 路径格式正确”的校验，不能证明 Windows RPA 机器已经同步完成；此时必须生成 Windows 端预检清单，供 Windows 端软件或 RPA 在执行前二次校验
- OneDrive、坚果云、Dropbox 等同步目录是否存在明显同步异常风险

路径校验不能仅依赖文件大小判断占位符。系统应采用“存在性检查 + 可读性探测 + 类型校验 + 目标端预检清单”的组合策略。

### 7.4.1 目标端预检清单

每次导出任务单时，软件同时生成 `preflight.json`，记录本次任务单引用到的所有相对路径、目标平台绝对路径、文件大小、mtime、校验状态和生成设备信息。

当 Windows 端软件或 RPA 执行任务前，应读取 `preflight.json` 并在 Windows 本机完成二次校验。只有 Windows 端也能真实读取视频与封面文件时，才视为路径链路完整。

### 7.5 软件与 RPA 的文件写入边界

软件与 RPA 不同时写同一个文件。

- 软件负责生成任务单文件。
- RPA 只读取任务单文件。
- RPA 执行完成后写入独立状态 JSON 文件。状态文件采用临时文件写入后原子重命名的方式落盘，避免软件读到半写入文件。
- 软件读取状态 JSON 文件并更新数据库状态。
- 状态文件保留为执行记录，不因重试而删除。

该协议用于避免 Excel 文件锁、同步冲突和跨设备双写问题。

## 8. 任务单系统

### 8.1 任务单定位

任务单是每天的发布计划。任务单中的每一行对应一次 RPA 上传和发布操作。

同一个视频可以发布到多个平台。任务单按“视频 × 平台”展开。示例：每天选择 5 个视频并发布到 4 个平台时，任务单生成 20 行。

### 8.2 任务单数据来源

任务单由以下数据汇总生成：

- 视频库
- 标签库
- 标题库
- 平台账号配置
- 商品信息
- SKU 信息
- 封面路径
- 定时发布时间配置

### 8.3 任务单字段

任务单包含以下核心字段：

| 字段       | 说明                   |
| -------- | -------------------- |
| 任务 ID    | 唯一任务标识               |
| 任务日期     | 发布计划所属日期             |
| 定时发布时间   | RPA 或平台定时发布使用的时间     |
| 发布平台     | 抖音、小红书等具体平台          |
| 平台账号名称   | 当前平台下的账号名称           |
| SKU 款式名  | 商品款式名称               |
| SKU 编码   | 商品或视频对应的 SKU 编码      |
| 商品名称     | 商品名称                 |
| 视频文件名    | 本地视频文件名              |
| 视频本地路径   | 导出时拼接后的目标系统绝对路径      |
| 封面本地路径   | 导出时拼接后的目标系统绝对路径      |
| 标题       | 从标题库选择或生成的标题         |
| 标签一至标签五  | 从标签库匹配或按策略分配的标签      |
| 任务状态     | 待执行、执行中、成功、失败等状态     |
| RPA 回写信息 | RPA 执行后的状态、错误信息和时间信息 |

影刀 RPA 字段格式由影刀流程后期适配，软件内部字段可以先按产品需要自由命名。导出字段支持中文和英文两种命名风格。

### 8.4 触发方式

任务单支持手动模式和定时模式。

| 模式   | 规则                         |
| ---- | -------------------------- |
| 手动模式 | 用户设置筛选条件、数量配比、平台账号后点击生成    |
| 定时模式 | 系统保留既有筛选配置，到点自动生成并保存到任务单目录 |

### 8.5 视频筛选策略

任务单生成时支持以下视频筛选倾向：

- 热门 SKU 倾向
- 低发倾向
- 近期热点
- 自定义

### 8.6 数量与平台展开

用户配置每天发布的视频数量，并勾选发布平台和平台账号。系统根据视频数量与平台数量自动展开任务行。

系统排除已经发布过的“视频 × 平台”组合，避免重复发布同一组合。

### 8.7 标题策略

任务单生成时支持三种标题选择策略：

| 策略    | 说明                |
| ----- | ----------------- |
| 爆款倾向  | 优先选择爆款分数较高的标题     |
| 新标题测试 | 优先选择新生成或未充分测试的标题  |
| 随机自定义 | 按用户自定义规则或随机方式选择标题 |

标题使用后，系统回写标题使用次数。

### 8.8 标签策略

标签库由用户手动维护。每个 SKU 可以配置默认标签和测试标签。

任务单生成时支持按比例分配默认标签和测试标签，用于标签测试。

### 8.9 发布时间分配

任务单发布时间采用时间锚点加随机抖动的方式生成。

用户可按每天发布的视频数配置 N 个时间锚点。每个锚点不是固定时间，而是在锚点附近 ±15 分钟范围内随机生成具体发布时间。

示例：锚点为 9:00 时，系统可生成 8:55–9:15 之间的随机发布时间。

当任务行数量不能被时间锚点数量整除时，系统必须保留全部任务行，不得因为整除取余丢失发布时间。系统按 round-robin 方式把余数分配到前若干锚点，并对最终时间排序。

### 8.10 任务单编辑与重新生成

任务单生成后支持人工编辑和重新生成。

任务单编辑能力包括：

- 行内编辑
- 批量更换标题
- 批量调整标签
- 修改发布时间
- 删除任务行
- 增加任务行
- 重新生成任务单

### 8.11 历史任务单

任务单支持按日期切换和查看历史记录。

### 8.12 导出格式

任务单支持三种导出格式：

- Excel
- CSV
- JSON

### 8.13 RPA 状态回写

RPA 软件为 Windows 端影刀。短视频运营工作台本身在 macOS 与 Windows 两端功能一致；RPA 是外部 Windows 自动化执行器，不代表软件功能在 Windows 端更多。

任务单导出后，影刀 RPA 逐行读取并执行发布任务。RPA 每完成一行任务，就在当前任务单日期目录下的 `status/` 子目录写入一个小型 JSON 状态文件。

状态文件示例：

```text
tasks/2026-05-07/status/T20260507-001__attempt-001.json
```

状态文件必须包含 `task_id`、`attempt_no`、`run_key`、`status`、`executed_at`、`error_code`、`error_message` 等字段。`run_key` 由 `task_id + attempt_no` 组成，用于幂等处理。

软件监听或定时扫描 `status/` 目录，并将 RPA 状态回写到本地数据库。默认扫描间隔为 5 秒。软件基于 `run_key` 幂等处理状态文件；同一状态文件因云盘重复触发时不得重复增加标题使用次数或重复更新副作用。

失败任务重试时，软件不删除旧状态文件，而是生成新的 `attempt_no` 和新的 `run_key`。旧状态文件保留为历史执行记录。

### 8.14 状态变更副作用

当任务状态被确认完成后，系统自动执行以下副作用：

- 对应“视频 × 平台”组合标记为已使用或已发布
- 对应标题的使用次数 `use_count` 增加
- 任务单状态更新为成功或失败
- 错误信息写入日志和任务记录

## 9. 数据库系统

### 9.1 数据库总体范围

v1 业务数据库包含六类核心业务库：

- 视频库
- 标签库
- 标题库
- 图片提示词库
- 图片库
- 视频文案库

除六类业务库外，系统还包含任务单、API 调用日志、定时任务执行记录、备份记录和应用配置等数据表。

### 9.2 通用数据库能力

所有主要数据库页面支持：

- 搜索
- 筛选
- 排序
- 表格视图
- 网格视图
- 批量操作
- 批量入库
- 批量软删
- 批量加入反面库

大列表必须使用虚拟列表技术，保证滚动性能。

### 9.3 视频库

视频库索引本地文件夹，不保存视频文件本体。

视频库记录以下信息：

- 视频文件名
- 视频相对路径
- 当前系统绝对路径展示
- SKU 编码
- SKU 款式
- 品牌
- 封面路径
- 视频使用状态
- 视频与平台组合的发布状态
- 辅助信息字段

视频库支持三种入库和更新方式：

- 实时监听
- 定时扫描
- 手动重扫

视频扫描时，系统使用 ffmpeg 抽取首帧作为缩略图。视频 hover 时支持播放前 3 秒预览。

### 9.4 标签库

标签库由用户手动维护。标签库按 SKU 关联视频库。

标签库记录以下信息：

- SKU 编码
- SKU 款式名
- 标签一
- 标签二
- 标签三
- 标签四
- 标签五
- 标签组类型

每个 SKU 可配置多套标签，包括默认标签和测试标签。

标签库支持 CSV 批量导入。

### 9.5 标题库

标题库用于管理短视频发布标题。

标题库记录以下信息：

- 标题内容
- 所属 SKU 或适用范围
- 使用状态
- 爆款分数
- 使用次数
- 创建来源
- 创建时间
- 更新时间

标题主要从标题库中选择并写入任务单。标题库支持定时生成新标题。任务单生成时，系统按标题策略随机或自定义选中标题并放入任务单。

标题被任务单使用后，系统自动回写使用次数。

### 9.6 图片提示词库

图片提示词库是独立数据库，与图片库分离。

图片提示词库记录以下信息：

- 提示词内容
- 所属场景
- 来源 Skill
- 创建时间
- 已生成图数
- 入库图数
- 入库率
- 状态
- 备注
- 标签

提示词状态包括：

- 草稿
- 优质
- 反面
- 待审

入库率由系统自动计算。每次图片入库或软删时，系统回写对应提示词的已生成图数、入库图数和入库率。

### 9.7 图片库

图片库用于管理 AI 生成图片和后续筛选保留的图片素材。

图片库记录以下信息：

- 图片文件路径
- 所属工作空间
- 所属场景
- 来源提示词
- 来源 Skill
- 生成模型
- 图片比例
- 创建时间
- 入库状态
- 软删状态

图片库与图片提示词库支持双向溯源。

### 9.8 视频文案库

视频文案库用于保存视频文案生成结果。

视频文案库 v1 只做基础 CRUD，不与任务单强制关联。

视频文案产出物包括：

- 口播脚本
- 剪映文字转语音文案

视频文案 v1 基于 Skill 凭空生成，不要求基于已有视频分析生成。

## 10. Skill 系统

### 10.1 Skill 定义

Skill 是内置系统提示词包，用于承载规范性提示词、流程提示词、模板、示例、注意事项、禁止事项和自我审查规则。

AI 调用时的完整提示词由 Skill 展开内容和本次任务提示词组成。

```text
完整提示词 = 已展开 Skill 内容 + 本次任务提示词
```

本次任务提示词是可选项。当任务提示词为空时，系统按 Skill 内置流程直接运行。当任务提示词非空时，任务提示词作为额外指令拼接到 Skill 后面。

### 10.2 Skill 中心

Skill 中心是全局模块，跨工作空间共享。

Skill 中心负责：

- Skill 浏览
- Skill 管理
- Skill 编辑
- Skill 测试
- Skill 版本管理
- 官方 Skill 同步
- 官方 Skill 复制
- 自建 Skill 创建

定时生成任务不放在 Skill 中心配置。定时生成任务配置在各自的工作流页面中。

### 10.3 Skill 与工作空间关系

Skill 池本体全局存在。每个工作空间通过 `skills_config/` 独立记录启用哪些 Skill。

不同工作空间可以启用不同 Skill，也可以为相同工作流选择不同 Skill。

### 10.4 Skill 类型

Skill 分为三类：

| 类型       | 来源        | 可编辑  | 可复原       | 接收云端更新 |
| -------- | --------- | ---- | --------- | ------ |
| 官方原版     | 云端拉取      | 否，只读 | 不适用       | 是      |
| 官方副本     | 用户从官方原版复制 | 是    | 是，回到复制时版本 | 否      |
| 自建 Skill | 用户原创      | 是    | 否         | 否      |

官方原版只读。用户要修改官方 Skill 时，必须先创建官方副本。

官方副本带血缘元数据，记录来源官方 Skill 及复制时版本。官方副本的复原是回到复制时版本，不是回到最新官方版。

官方副本提供“升级到最新官方版”选项。升级会覆盖当前修改，并需要二次确认。

自建 Skill 不接收官方更新，也不提供复原到官方版本的能力。

### 10.5 Skill 物理结构

一个 Skill 是一个文件夹。文件夹内包含 `meta.json`、`SKILL.md` 主文件和若干 `.md` 子文件。

示例：

```text
title_skill_brand_a/
├── meta.json
├── SKILL.md
├── brand_info.md
├── product_info.md
├── examples_good.md
├── examples_bad.md
├── angle_strategies.md
└── self_review_rules.md
```

### 10.6 include 拼接机制

`SKILL.md` 支持使用 `{{include: xxx.md}}` 引用子文件。

示例：

```md
{{include: brand_info.md}}
{{include: product_info.md}}
{{include: examples/good.md}}
{{include: self_review_rules.md}}
```

软件在调用大模型前执行字符串替换和静态展开。LLM 收到的是已经展开后的纯文本，不需要也不会自行读取本地文件。

### 10.7 include 约束

include 机制支持子文件夹路径，例如：

```md
{{include: examples/good.md}}
```

系统检测循环引用并报错。

最大递归深度为 5。

include 解析必须运行在 Skill 根目录沙箱内。引用路径不得通过 `../`、绝对路径或符号链接跳出当前 Skill 文件夹；不得引用隐藏目录、`.snapshots/`、非 Markdown 文件或超过大小限制的文件。出现越界、缺失、循环引用或递归过深时，系统必须报错并中止本次 LLM 调用。

### 10.8 Skill 多样性机制

Skill 内部必须支持多样性策略，用于降低长期自动生成内容的趋同风险。

多样性策略可以通过以下子文件实现：

- 场景切入库
- 身份切入库
- 角度策略库
- 正面示例库
- 反面示例库
- 自我审查打分规则

### 10.9 Skill 编辑器

Skill 中心内置 Markdown 编辑器。

编辑器支持：

- include 标记高亮
- include 文件点击跳转
- 保存自动快照
- 版本还原
- 即时测试

即时测试通过弹层完成。用户输入测试提示词、选择模型后立即调用并查看输出。即时测试结果不自动入库。

### 10.10 官方 Skill 云端同步

官方 Skill 由软件开发者在云端维护。

云端官方 Skill 采用公开 GitHub 仓库管理。仓库可公开访问，不在客户端内置私有仓库 token。

软件通过 manifest 拉取官方 Skill 索引和版本信息。启动时最多每天检查一次 manifest。发现更新后提示用户。manifest 应包含每个 Skill 包的版本、文件清单和 hash；安装或升级 Skill 时必须校验下载内容完整性。

离线状态下使用本地缓存。

## 11. AI 工作流通用规则

标题工作区、视频文案工作区和图片提示词生成共享同一套 AI 调用模式。

通用调用模式为：

```text
选择 Skill + 输入任务提示词（可选）+ 选择模型 → 流式输出 → 审阅 → 入库或丢弃
```

### 11.1 多模型对比

系统必须支持多模型对比。

多模型对比时，不同模型并发调用，结果分列展示。某一家模型失败不影响其他模型继续输出。

### 11.2 流式输出

AI 文本生成必须支持流式输出，不能等完整结果全部生成完才显示。

### 11.3 取消能力

用户可以中途取消正在进行的生成任务。

### 11.4 失败容错

单家模型调用失败时，系统记录失败信息，并保留其他成功模型的输出。

### 11.5 调用日志

每次 AI 调用写入 `api_call_log` 表，用于费用统计、调用追踪和排障。

### 11.6 入库评分

标题入库时弹层允许用户填写爆款分数。爆款分数为可选项。

## 12. 标题生成工作区

标题生成工作区用于批量生成短视频标题。

标题生成工作区支持：

- 选择 Skill
- 输入任务提示词
- 空任务提示词运行
- 选择一个或多个模型
- 多模型对比
- 流式输出
- 结果审阅
- 批量入库标题库
- 批量软删
- 批量加入反面库
- 入库时填写爆款分数
- 定时生成标题

标题库可以定时生成新标题。标题生成的定时任务配置入口位于标题工作区，不放在 Skill 中心。

## 13. 图片工作室

### 13.1 图片工作室定位

图片工作室用于批量生成图片提示词、管理图片提示词、调用图片模型生成图片，并对生成图片进行审阅、入库、软删和溯源。

图片生成的用途包括：

- 视频封面
- 直播封面
- 主图素材
- SKU 素材
- 详情页素材
- 图片转文字的拼接素材
- 其他可扩展图片场景

### 13.2 整体结构

图片工作室采用 3-tab 复合页面：

1. 提示词生成
2. 提示词库
3. 图片生成

### 13.3 场景预设

图片工作室顶部提供场景预设 tab。

默认场景包括：

- 视频封面
- 直播封面
- 主图
- SKU 素材
- 详情页
- 拼接
- 新增自定义场景

每个场景预设包含：

- 关联 Skill
- 默认图片比例
- 默认生成数量
- 默认输出文件夹
- 默认调用模型

### 13.4 提示词生成 tab

提示词生成 tab 只调用文本 LLM，不调用图片 API。

工作流程为：

```text
选择场景预设 → 输入种子描述 → 选择 Skill 和模型 → 批量生成图片提示词 → 审阅 → 入提示词库
```

提示词来源包括：

- Skill
- 用户输入的种子提示词
- AI 基于种子提示词扩展生成的提示词

### 13.5 提示词库 tab

提示词库 tab 是标准数据库管理页。

提示词库支持：

- 搜索
- 筛选
- 排序
- 按入库率排序
- 状态管理
- 批量操作
- 选中提示词生成图片

### 13.6 图片生成 tab

图片生成 tab 调用图片 API。

工作流程为：

```text
选择提示词 → 选择图片模型 → 设置比例 → 设置数量 → 调用图片 API → 流式或逐张出图 → 网格审阅 → 入库或软删
```

### 13.7 双向溯源

图片和提示词之间必须支持双向溯源。

图片卡片悬浮时提供“跳转到提示词”操作。用户点击后，系统自动切换到提示词库 tab，并高亮定位该图片对应的提示词。

### 13.8 图片软删与反面库

图片删除采用软删机制。

软删图片移动到 `_trash/`，并进入反面图库或反面数据状态。

系统支持在设置中配置软删保留期。默认可选项包括：

- 30 天
- 60 天
- 90 天
- 自定义
- 永不自动清理

超过保留期后，系统执行真实删除。

当某一条提示词生成的图片全部被软删时，系统建议将该提示词标记为“反面”。

反面图库和反面提示词用于后续 AI 总结学习和优化 Skill。v1 先收集数据，不实现 AI 自动总结优化 Skill。

## 14. 视频文案生成工作区

视频文案生成工作区用于生成短视频口播脚本和剪映文字转语音文案。

v1 中，视频文案生成不与任务单强制关联。

视频文案生成采用与标题生成类似的模式：

```text
选择 Skill + 输入任务提示词（可选）+ 选择模型 → 流式生成 → 审阅 → 入文案库
```

视频文案 v1 以凭空生成为主，根据 Skill 中的品牌介绍、商品介绍、功能介绍、风格要求和文案规范生成。

视频文案生成结果进入独立视频文案库。

## 15. 封面管理

### 15.1 封面管理定位

封面管理用于查看本地视频文件，并为短视频批量选择、裁剪、保存和管理封面。

封面管理需要支持一次性处理几十个甚至上百个视频。

### 15.2 v1 操作模式

v1 采用人工手动拉时间轴选择封面。

时间轴必须支持 hover 即时预览，不能只在拖动时预览。

系统使用 ffmpeg 预先抽取 30–60 帧缩略图序列并缓存，提升时间轴预览性能。

### 15.3 默认封面

如果视频未手动设置封面，系统使用首帧作为默认封面，并截图保存到 `covers/` 目录。

系统将封面相对路径写入数据库中的 `cover_relative_path` 字段。

### 15.4 封面比例

封面截图支持自定义比例。

默认比例包括：

- 3:4
- 9:16
- 1:1
- 自定义比例

比例裁剪使用半透明遮罩，遮罩区域可拖动。

### 15.5 封面保存规则

封面保存到独立 `covers/` 文件夹，并镜像 `videos/` 结构。

文件命名规则为：

```text
视频名__比例.jpg
```

示例：

```text
style_a__3x4.jpg
style_a__9x16.jpg
```

一个视频允许保存多个比例封面。

### 15.6 封面管理视图

封面管理支持列表视图和网格视图切换。

### 15.7 批量能力

封面管理支持对未选择封面的视频批量应用首帧封面。

### 15.8 快捷键

封面管理支持键盘快捷键：

| 快捷键 | 动作    |
| --- | ----- |
| J   | 上一个视频 |
| K   | 下一个视频 |

### 15.9 v1 不实现的封面能力

v1 不实现 AI 推荐封面。

v1 不实现封面创作能力，例如加标题、滤镜、设计稿等。

AI 推荐封面和封面创作保留后续升级入口。

## 16. 定时任务系统

### 16.1 配置入口

定时任务配置入口位于各自工作流页面。

示例：

- 任务单页面配置定时生成任务单
- 标题工作区配置定时生成标题

总览页展示：

- 所有定时任务
- 下一次触发时间
- 启用状态
- 历史执行记录

- API 配额消耗

### 16.3 定时任务类型

- 任务单生成
- 标题生成
- 图片生成

- 补跑最近一次

- 跳过错过任务

### 16.6 执行历史

定时任务执行记录写入 `scheduled_job_runs` 表。

失败任务记录错误信息，下次仍按正常调度继续执行。

## 17. Provider 抽象层

### 17.1 LLM Provider

系统抽象 LLM Provider 接口，支持多家文本模型服务。

v1 规划支持：

- Anthropic
- OpenAI
- Google
- 后续可扩展 Provider

### 17.2 图片 Provider

系统抽象图片 Provider 接口，支持多家图片生成服务。

v1 规划支持或预留：

- GPT Image 2
- 即梦
- 可灵
- 后续可扩展图片模型

### 17.3 API key 管理

API key 存放在全局凭证池。

每家 Provider 支持保存多份 API key。每份 key 可命名，并可设置默认 key。

每个工作空间可以指定该工作空间优先使用哪个模型或 key。

### 17.4 API key 加密

API key 必须加密存储。默认模式下，macOS 使用系统钥匙串或 Electron `safeStorage`，Windows 使用 Windows DPAPI / Credential Manager 或 Electron `safeStorage`。默认加密结果绑定当前系统用户和设备，不保证能直接在另一台电脑解密。

为了支持多工作电脑无缝切换，软件可提供可选的“跨设备凭证保险箱”。该保险箱使用用户自设主密码或恢复密钥进行跨平台加密，文件可放在云盘同步目录中。启用后，API key 可在 macOS 与 Windows 间同步，但用户必须理解：保险箱安全性取决于主密码强度；丢失主密码无法恢复；保险箱文件不应写入日志和反馈包。

工作空间内容、品牌数据、业务数据库、用户自建 Skill、工作空间启用配置和任务单可通过云盘同步。缓存、缩略图、临时文件和普通日志默认不参与同步。若同一工作空间在两台电脑上同时打开，软件必须通过锁文件和心跳检测阻止双写，或把后打开的一端切换为只读模式。

### 17.5 调用能力

Provider 抽象层支持：

- 自动重试
- 可配置重试次数
- 指数退避
- AbortController 取消
- 多模型并发调用
- `Promise.allSettled` 容错模式

单个 Provider 失败不影响其他 Provider 的调用结果。

## 20. 技术方向

### 20.1 最终技术栈

| 层         | 选择                       |
| --------- | ------------------------ |
| 桌面壳       | Electron                 |

| 表格读写      | exceljs                  |
| GitHub 同步 | Octokit                  |
| 打包        | electron-builder         |

### 20.2 Electron 使用纪律

开发时必须遵守以下约束：

1. 严格区分主进程、渲染进程和后台任务职责。
2. IPC 通信必须全部 TypeScript 类型化。
3. 超过 100 行的数据列表使用 `react-window` 或同类虚拟列表方案。
4. 视频预览不做实时高频 scrubbing，使用 ffmpeg 预渲染缩略图序列。
5. SQLite 使用 `better-sqlite3`。
6. 关闭 `nodeIntegration`。
7. 使用 `contextBridge` 暴露安全 API。
8. 启用 ASAR 打包。

### 20.3 UI/UX 文档与技术选型边界

UI/UX 文档本身不承载完整技术选型论证。

交给原型设计工具或 Claude design 的说明中必须明确：

- 这是桌面应用，不是网页应用。
- 应用需要支持 Mac 和 Windows。
- 原型可用 HTML/React 静态原型表达。

## 23. 术语定义

| 术语       | 定义                                     |
| -------- | -------------------------------------- |
| 工作空间     | 一个品牌的独立运营空间，包含业务库、任务单、账号清单和 Skill 启用配置 |
| Skill    | 内置系统提示词包，包含主文件、子文件和元数据                 |

| 自建 Skill | 用户原创的 Skill                            |
| 任务单      | 每天的发布计划                                |
| 任务行      | 任务单中的一行，对应一次“视频 × 平台”发布操作              |
| 状态文件     | RPA 完成任务后写入的 `<任务ID>.json` 文件          |

| 入库率      | 提示词产生的入库图数 / 已生成图数                     |
| 路径映射     | Mac 根路径与 Windows 根路径之间的对应关系            |

## 24. 最终产品结构摘要

短视频运营工作台采用“工作空间 + 数据库 + 工作流 + Skill + Provider + RPA 协议”的结构。

工作空间负责品牌隔离。数据库负责管理视频、标签、标题、提示词、图片和文案。工作流负责生成任务单、标题、图片、文案和封面。Skill 负责沉淀提示词规范和生成流程。Provider 抽象层负责接入不同文本模型和图片模型。RPA 协议负责把任务单稳定交给 Windows 影刀执行，并通过独立状态文件回写执行结果。

v1 的核心目标是稳定完成本地内容管理、AI 内容生产、任务单生成、跨平台路径导出和 RPA 状态回写。后续版本再扩展协作、AI 推荐封面、封面创作、复杂文案工作流、后台守护进程和自动优化 Skill 等能力。
